## 1. 引言：内容工厂

欢迎来到 GuluCoding 教育创新的核心：**内容工厂**。这个新模型从根本上改变了我们创建和交付课程的方式。

**旧方式:** 每一门新课程都必须由工程师编码实现。这个过程缓慢、昂贵，而且让你完全依赖工程排期来修正一个错别字或更改一个数字。

**新方式 (内容工厂):** 我们的应用程序现在是一个“游戏引擎”，就像一个智能的电子游戏机。而课程本身则是“游戏卡带”，它们是用一种叫做 **JSON** 的格式编写的简单文本文件。

**你的角色:** 现在，你们就是创造者！你们只需通过创建和编辑这些 JSON 文本文件，就可以设计、构建、测试和发布新课程。你再也无需等待工程师。本文档就是你掌握这一强大新工作流程的指南。

## 2. 理解“游戏卡带”：`level.json` 文件

GuluCoding 中的每一节课都由一个 `level.json` 文件定义。你可以把这个文件想象成一个详细的食谱，它精确地告诉 Gulu 游戏引擎该做什么。

一个 `level.json` 文件包含三个主要部分：

### 2.1 `meta`: 课程的“身份证”

这部分包含了关于课程的基本信息。

*   `title`: 课程的名称 (例如, "交换两个变量")。
*   `description`: 关于学生将学到什么的简短说明。
*   *我们同时提供了 `en` 和 `cn` 字段，以便轻松翻译。*

### 2.2 `layout`: 布置舞台

这部分描述了课程开始时学生在屏幕上看到的所有内容。

*   `elements`: 所有视觉元素的列表。每个元素都有：
    *   `id`: 你给它取的一个唯一名字 (例如, `"box_a"`)。
    *   `type`: 它是什么类型的元素 (例如, `"VariableBox"`)。
    *   `position`: 它在屏幕上的 `x` 和 `y` 坐标。
    *   `initialValue`: 它开始时带有的数字或文本。

**示例:** 要创建一个名为 "a"、值为 10 的变量盒子，你可以这样写：
```json
{
  "id": "box_a",
  "type": "VariableBox",
  "position": { "x": 150, "y": 250 },
  "initialValue": 10
}
```

### 2.3 `logic`: 游戏的“分步规则”

这是最重要的部分。它是一个步骤列表，定义了课程的整个互动流程。引擎在任何时候只关注一个步骤。

每个步骤都有三个关键部分：

1.  **`instruction`**: 显示给学生的文本 (例如, "把数字 10 拖进盒子 'a'")。
2.  **`trigger`**: 学生为完成该步骤必须执行的精确动作。
    *   **Type:** 是什么类型的动作？ (例如, `ButtonClick`, `DragDrop`)。
    *   **Target:** 涉及哪些元素？ (例如, `sourceId: "value_10"`, `targetId: "box_a"`)。
3.  **`onSuccess`**: 学生执行正确动作后会发生什么。
    *   `playSound`: 播放什么声音 (例如, `"ding"`, `"win"`)。
    *   `nextStep`: 接下来进入哪个步骤。
    *   `action` (可选): 执行一个动画或变更 (例如, 让一个新盒子出现)。

通过将这些步骤串联起来，你可以创建出具有分支逻辑、提示和奖励的复杂且引人入Engaging的课程。

## 3. 内容创作工作流

现在，创建一个新课程是一个简单的四步过程。

### 第一步: 设计课程 (在纸上或白板上)

在接触电脑之前，先规划好你的课程：
*   学习目标是什么？
*   需要哪些视觉元素？
*   给学生的分步指示是什么？
*   每一步的正确动作是什么？

### 第二步: 编写 `level.json`

使用文本编辑器（如 VS Code）或我们未来的“课程编辑器”工具，将你的设计转换成一个 `level.json` 文件。从一个现有课程的模板开始，然后进行修改。

*   定义你的 `meta` 信息。
*   设置初始的 `layout` 和 `elements`。
*   构建 `logic.steps` 数组，仔细定义每一步的 `instruction`, `trigger`, 和 `onSuccess`。

### 第三步: 在沙盒中测试

这是至关重要的一步。导航到我们的内部“沙盒”页面 (例如, `http://localhost:3000/lab/engine`)。

*   将你完整的 `level.json` 内容粘贴到页面的编辑器中。
*   屏幕右侧将立即为你显示一个可实时游玩的课程预览。
*   像学生一样玩一遍你的课程。
*   如果你发现一个 bug 或想改变某些东西，只需编辑 JSON 文本，预览就会实时更新。

这使你能够完善你的课程，而无需等待工程师或新的应用部署。

### 第四步: 提交审查

当你在沙盒中对你的课程感到满意后，保存你的 `level.json` 文件并将其提交到指定的内容仓库或文件夹。然后它会经过一个快速的 QA 检查，之后便会发布给所有用户。

## 4. 产品功能与路线图 (面向用户)

这个新架构解锁了许多新功能，这些功能现在已经加入我们的路线图。

### 核心产品功能
*   **用户账户:** 学生可以注册、登录并保存他们的学习进度。
*   **学习仪表盘:** 一个展示已完成课程、获得的星星以及下一个要挑战的课程的中心枢纽。
*   **多学科访问:** 一次订阅即可访问我们所有的学科 (C++, ESL, 数学等)。
*   **AI 导师:** “解释这个”功能，现在由安全的后端提供支持，提供安全有益的指导。

### 近期路线图 (未来6个月)
*   **第一季度: “成功之路”改造:** 所有现有的 C++ 课程将被迁移到新引擎，创造一个无缝、可持久化的学习路径。
*   **第一季度: Pro 订阅上线:** 正式推出我们的付费订阅服务，用户可访问所有功能。
*   **第二季度: ESL 模块上线:** 基于同一引擎构建的第一套互动式英语课程。
*   **第二季度: 家长/教师门户:** 一个供家长和教师追踪学生学习进度和表现的仪表盘。
*   **第三季度: 数学模块上线:** 第一套互动式数学课程。
