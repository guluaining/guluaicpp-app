# 附件一：GuluCoding 产品现状评估与下一阶段产品化立项建议书
**Attachment 1: MVP Status Report & Product Proposal**

## 第一部分：当前 MVP 阶段性总结报告 (Current Status Report)

### 1.1 项目背景与目标
*   **初衷：** 解决 K12 阶段 C++ 学习门槛高、抽象枯燥、容易放弃的痛点。
*   **MVP 目标：** 验证“可视化交互 + AI 导师 + 游戏化隐喻”教学法的有效性。
*   **当前状态：** 已完成 Web 端 MVP 开发并部署，具备核心演示功能。

### 1.2 核心成果 (Achievements)
*   **产品力验证：** 成功实现了变量赋值、交换、找最大值、排序等核心算法的**可视化交互引擎**。
*   **教学法创新：** 确立了“Gulu 游戏模式”与“专业模式”双轨制，独创“盒子/贴纸/替身”教学隐喻。
*   **AI 融合：** 跑通了 Gemini API 实时代码解释功能，实现了多语言（中/英）无缝切换。
*   **内容板块：** 完成了“成功之路”演示模块，初步构建了从双语到国际化的理论体系。

### 1.3 现存局限性与风险 (Limitations & Risks)
*   **架构短板：** 纯前端架构 (SPA)，无后端、无数据库。**用户数据无法保存**（刷新即丢失），无法积累用户资产。
*   **安全隐患：** AI API Key 暴露在前端，存在被盗用和资损风险。
*   **合规风险：** 直接调用 Google 服务，导致**中国大陆用户无法访问**，且未进行 ICP 备案，无法在中国合法运营。
*   **商业化缺失：** 缺乏账户体系、支付接口和权限控制，无法转化为收入。
*   **工程质量：** 快速迭代导致代码存在“技术债”，部分 UI/交互在特定设备（如 iPad）上存在兼容性 bug。

## 第二部分：下一阶段产品化立项建议 (Next Step Product Proposal)

### 2.1 战略定位
*   **产品定义：** 全球首个**AI 驱动的双语 C++ 可视化学习 SaaS 平台**。
*   **市场策略：** **“一个内核，双轨运营”**。
    *   **国际版 (Global)：** 面向北美/欧洲/东南亚，通过 Google/Stripe 生态变现。
    *   **中国版 (CN)：** 面向国内市场，符合双减政策（素质教育/科技特长），完全合规（ICP/网信办），接入微信/支付宝生态。

### 2.2 核心技术架构重构 (Architecture 2.0)
*   **全栈迁移：** 从 Vite 迁移至 **Next.js (Monorepo)** 架构。
*   **后端建设：**
    *   建立 **PostgreSQL 数据库**：存储用户进度、班级信息、答题记录。
    *   建立 **API 中转层**：后端代理 AI 请求，隐藏 Key，并根据用户地区分流（Global 调 Gemini，CN 调通义千问/DeepSeek）。
*   **账户体系 (Auth)：**
    *   Global：Email, Google Login.
    *   CN：微信扫码登录，手机号一键登录（实名认证）。

### 2.3 功能模块规划 (Feature Roadmap)

| 模块 | MVP 现状 | 产品化目标 (V1.0) |
| :--- | :--- | :--- |
| **用户系统** | 无 | 注册/登录、个人中心、学习仪表盘、多角色（学生/家长/老师） |
| **数据持久化** | 刷新丢失 | 云端同步进度、错题本、学习时长统计、成就徽章系统 |
| **AI 导师** | 前端直连 | 后端代理、额度控制（Free/Pro）、敏感词过滤（CN） |
| **课程体系** | 仅算法演示 | 完整的关卡地图（解锁制）、ESL 专项训练营、配套视频课 |
| **支付变现** | 无 | 订阅制 (SaaS)、单课购买、企业/学校采购通道 |
| **设备兼容** | iPad 适配不佳 | 完美适配 PC/Mac/iPad/Android 平板，触摸交互优化 |

### 2.4 商业模式设计 (Business Model)
*   **C 端 (ToC)：**
    *   **Freemium (免费增值)：** 免费试玩前 3 关 -> 注册保存进度 -> 订阅 Pro 会员解锁 AI 无限问答及高阶算法。
*   **B 端 (ToB)：**
    *   **学校/机构版：** 教师管理后台、批量账号授权、班级进度监控、一键布置作业。

### 2.5 实施路线图 (Execution Timeline)

*   **阶段一：基础设施与合规 (第 1-2 个月)**
    *   注册国内公司，申请域名备案，服务器采购。
    *   搭建 Next.js 框架，完成数据库设计，实现中美数据隔离。
*   **阶段二：账户与数据 (第 3 个月)**
    *   完成登录注册系统，实现数据云端保存。
    *   修复现有 MVP 中的交互 Bug（特别是移动端）。
*   **阶段三：支付与 AI 路由 (第 4 个月)**
    *   接入 Stripe 和 微信支付。
    *   实现 AI 模型的中美自动切换与流式输出。
*   **阶段四：正式发布 (第 5 个月)**
    *   内容填充（完善七步路线图课程）。
    *   灰度测试，正式上线运营。

### 2.6 资源需求预估
*   **人力：** 全栈工程师 (1-2人)、UI/UX 设计 (0.5人)、教研内容 (1人)。
*   **云服务成本：** 预估初期 $200-$500/月 (AWS + 阿里云)。
